DatePicker — CLDR race condition

Reproduces the bug where getCachedLocaleDataInstance creates a LocaleData instance before CLDR data has finished loading, leaving mData undefined and crashing on first render.

Scenario 1 — setLanguage("bg") then immediately mount

Calls setLanguage("bg") without awaiting it, then immediately adds a <ui5-date-picker>. The race: the component renders before fetchCldr("bg") resolves, so mData is undefined → crash.
